## Пакет context в Go

**Пакет `context`** предназначен для управления жизненным циклом операций, передачи сигналов отмены (cancellation), установки дедлайнов (deadlines/timeouts), а также для проброса метаданных (request-scoped values) по всему дереву вызовов внутри горутин.

Основная цель `context` — эффективное управление ресурсами и предотвращение утечек горутин (goroutine leaks) при выполнении долгих операций (I/O, сетевые запросы, запросы к БД).

---

### Ключевые области применения

1. **Каскадная отмена (Cascading Cancellation):** При отмене родительского контекста все дочерние контексты и связанные с ними операции отменяются автоматически.
2. **Управление таймаутами (Timeouts & Deadlines):** Автоматическая генерация сигнала отмены по истечении жестко заданного времени выполнения (`WithTimeout`, `WithDeadline`).
3. **Передача scoped-данных (Request-scoped Values):** Проброс сквозных метаданных (Trace ID, Request ID, данные авторизации) для логирования и мониторинга.

---

### Иерархия и типы контекстов

Контексты строятся в виде **направленного дерева**, где каждый дочерний контекст оборачивает родительский:

- **`context.Background()`** — корневой пустой контекст. Используется на самом верхнем уровне (в `main`, в обработчиках запросов, в тестах).
- **`context.TODO()`** — пустой контекст-заглушка. Используется, когда финальная логика контекста еще не определена или код находится в процессе рефакторинга.
- **`WithCancel(parent)`** — возвращает дочерний контекст и функцию `cancel()`. Отменяется вручную при вызове `cancel()`.
- **`WithTimeout(parent, duration)` / `WithDeadline(parent, time)`** — отменяются автоматически по времени либо вручную через `cancel()`.
- **`WithValue(parent, key, val)`** — создает контекст для передачи данных. Не имеет функции отмены.

---

### Правила хорошего тона при работе с context (Best Practices)

1. **Первый аргумент:** Контекст всегда передается первым аргументом в функцию с именем `ctx` (`func DoSomething(ctx context.Context, ...)`).
2. **Не хранить в структурах:** Не следует сохранять контекст внутри полей структур (исключение — `http.Request`). Он должен передаваться явно.
3. **Обязательный дефер:** Функция `cancel()`, возвращаемая фабриками контекста, должна вызываться всегда (обычно через `defer cancel()`), иначе это приведет к утечке ресурсов таймера или памяти.
4. **Контекст для данных — не хранилище:** Через `WithValue` нужно передавать только метаданные запроса, но не опциональные параметры функций или глобальные зависимости. Ключи для `WithValue` должны быть кастомного (неэкспортируемого) типа во избежание коллизий.

---

### Пример использования: HTTP-запрос с таймаутом

```go
package main

import (
	"context"
	"fmt"
	"net/http"
	"time"
)

// fetchData выполняет HTTP-запрос, уважая жизненный цикл переданного контекста.
func fetchData(ctx context.Context, url string) {
	// Создаем новый запрос с привязкой к контексту
	req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
	if err != nil {
		fmt.Printf("ошибка создания запроса: %v\n", err)
		return
	}

	// Выполняем запрос с использованием DefaultClient
	resp, err := http.DefaultClient.Do(req)
	if err != nil {
		// Если контекст завершился по таймауту, Do(req) вернет ошибку context.DeadlineExceeded
		fmt.Printf("ошибка выполнения запроса: %v\n", err)
		return
	}
	defer resp.Body.Close()

	fmt.Printf("Запрос успешно выполнен. Статус: %s\n", resp.Status)
}

func main() {
	// Инициализируем корневой контекст с жестким таймаутом в 2 секунды
	ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)

	// defer гарантирует освобождение ресурсов контекста сразу после выхода из main
	defer cancel()

	url := "https://typicode.com"
	fetchData(ctx, url)
}
```
