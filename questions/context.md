`Что такое контекст в Go и для чего он используется:`
Контекст в Go - это структура, которая предназначена для передачи сроков выполнения, отмены и значений между функциями и процессами. Он сипользуется для управления длительностью операций и отмены, что особенно важно в конкуретных средах.

Применение:

1. `Отмена операций:` Позволяет отменить выполнение функции, если, например, пользователь закрыл приложение или истекло время ожидания.
   Пример: Использование `context.WithTimeout` для установки срока выполнения HTTP-запроса.

2. `Передача значений:` Может передавать метаданные, такие как идентификаторы запросов, которые могут быть полезны для логирования или мониторинга.
   Пример: Добавление пользовательского идентификатора в контекст для отслеживания.

3. `Управление временем жизни:` Позволяет контролировать время жизни операций, что помогает избежать утечек ресурсов.

`Пример:` Использование `context.Background` как корневого контекста для инициализации.

```go
package main

import (
	"context"
	"fmt"
	"net/http"
	"time"
)

// функция, выполняющая http-запрос с контекстом
func fetchData(ctx context.Context, url string) {
	// создаем нвоый запрос с контекстом
	req, err := http.NewRequestWithContext(ctx, "GET", url, nil)
	if err != nil {
		fmt.Println("Ошибка создания запроса", err)
		return
	}

	// выполяем запрос
	resp, err := http.DefaultClient.Do(req)
	if err != nil {
		fmt.Println("Ошибка запроса", err)
		return
	}
	defer resp.Body.Close()

	fmt.Println("Статус ответа", resp.Status)
}

func main() {
	// создаем контекст с таймаутом в 2 секунды
	ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
	defer cancel() // отменяем контекст после завершения функции

	fetchData(ctx, "https://jsonplaceholder.typicode.com/posts")
}
```
