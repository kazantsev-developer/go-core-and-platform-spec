`Что такое defer в Go и зачем он нужен:`
Это ключевое слово которое используется для отложенного выполнения функции о тех пор, пока не завершиться ее выполнение. Это позволяет гарантировать, что определенные действия выполнятся в конце работы этой функции, не зависимо от того как она завершилась (нормально или с ошибкой).

```go
package main

import (
	"fmt"
	"os"
)

func main() {
	file, err := os.Open("example.txt")
	if err != nil {
		fmt.Println(err)
		return
	}
	// Отложение закрытого файла
	defer file.Close()

	// Читенеи файла и выполнение других действий
}

// 2. Упарвление памятью:
// Позволяет безопасно освобождать ресурсы, даже если возникают ошибки
func process() {
	defer func() {
		fmt.Println("Cleaning up...")
		// Освобождение ресурсов
	}()

	// Логика обработки
}
```

Преимущества
`Гарантия выполнения:` Функциии, объявленные с defer всегда выполнятся в порядке обратном их объяслению.
`Читаемость кода:` Упрощает управление ресурсами, делая код более чистым и понятным.
