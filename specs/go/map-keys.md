## Ключи в map: типы данных и критерий сравнимости

**Критерий сравнимости:** Ключом в Go-мапе может быть абсолютно любой тип данных, для которого на этапе компиляции определена операция сравнения (`==` и `!=`). Рантайму Go необходимо сравнивать хэши и сами значения ключей, чтобы находить нужные бакеты и элементы внутри них.

Если тип не поддерживает операцию сравнения, компилятор выдаст ошибку: `invalid map key type`.

---

### Какие типы МОГУТ быть ключами (Comparable types)

1. **Все встроенные скалярные типы:** `int` (все виды), `uint` (все виды), `float32/64`, `string`, `bool`, `complex64/128`.
2. **Указатели (Pointers):** Сравниваются адреса в памяти, на которые они указывают.
3. **Каналы (Channels):** Сравниваются как указатели (равны, если созданы одним вызовом `make`).
4. **Интерфейсы (Interfaces):** Могут быть ключами, если динамический тип, который лежит внутри интерфейса, является сравнимым. _Внимание: если в рантайме туда попадет несравнимый тип (например, слайс), программа упадет в панику._
5. **Массивы (Arrays):** **Важно!** Массивы имеют фиксированную длину и являются сравнимыми, если сравнимы элементы внутри них (например, `[2]int` — может быть ключом).
6. **Структуры (Structs):** Могут быть ключами, если **все** поля структуры без исключения являются сравнимыми.

---

### Какие типы НЕ МОГУТ быть ключами (Incomparable types)

Существует всего три базовых типа, которые нельзя использовать в качестве ключей напрямую:

1. **Слайсы (`slice`):** Из-за динамической природы и того, что они являются ссылками на нижележащий массив.
2. **Мапы (`map`):** Запрещено рантаймом.
3. **Функции (`func`):** Функции в Go можно сравнивать только с `nil`.

---

### Критические особенности и подводные камни

- **Массив vs Слайс:** Код `make(map[[2]int]string)` скомпилируется, так как это массив фиксированной длины `int`. Код `make(map[[]int]string)` выдаст ошибку компиляции, так как это слайс.
- **Структуры с "сюрпризом":** Если добавить в структуру хотя бы одно несравнимое поле (слайс, мапу или функцию), вся структура автоматически становится несравнимой и больше не может быть ключом мапы.
- **Опасность Float в качестве ключа:** Формально `float32` и `float64` использовать можно. Но на практике это **категорически запрещено** из-за спецификации `IEEE 754` и существования значения `NaN` (Not a Number). Так как `NaN == NaN` всегда возвращает `false`, вы никогда не сможете извлечь значение из мапы по ключу `NaN`, что приводит к утечке памяти и потере доступа к данным.

---

### Примеры кода

#### 1. Валидные композитные ключи (Массивы и Структуры)

```go
package main

import "fmt"

type Point struct {
	X, Y int
}

func main() {
	// Массив в качестве ключа (ПОЛНОСТЬЮ ВАЛИДНО)
	arrayMap := make(map[[2]int]string)
	arrayMap[[2]int{1, 2}] = "массив работает"
	fmt.Println(arrayMap)

	// Структура в качестве ключа (ВАЛИДНО, так как поля X и Y — int)
	structMap := make(map[Point]string)
	structMap[Point{X: 10, Y: 20}] = "точка А"
	fmt.Println(structMap)
}
```

#### 2. Опасность интерфейсов в качестве ключей (Паника в рантайме)

```go
package main

func main() {
	interfaceMap := make(map[interface{}]string)
	interfaceMap["строка_сравнима"] = "ok" // Компилируется и работает

	// ВНИМАНИЕ: Скомпилируется, но упадет в ПАНИКУ в рантайме!
	// panic: runtime error: hash of unhashable type []int
	interfaceMap[[]int{1, 2}] = "panic"
}
```

---

### Резюме по оптимизации

Ключом мапы в Go может быть любой тип, поддерживающий операторы `==` и `!=`. При использовании структур в качестве ключей компилятор генерирует хэш-функцию, которая поочередно хэширует каждое поле, что создает дополнительный оверхед на CPU. Для максимальной производительности в критических путях кода в качестве ключей следует использовать простые типы (`string`, `int`) и избегать использования интерфейсов (`interface{}` / `any`), чтобы исключить риск рантайм-паники при попадании туда несравнимого слайса или мапы.
