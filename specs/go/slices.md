## Слайсы в Go (Slices)

**Слайс (срез)** — это динамическая структура данных, являющаяся дескриптором над фиксированным массивом. В отличие от массивов, размер слайса не входит в его тип и может динамически изменяться во время выполнения.

---

### Внутреннее устройство (Под капотом)

Слайс представляет собой заголовок (дескриптор) фиксированного размера (24 байта на 64-битной архитектуре) и описан в исходном коде рантайма (`runtime/slice.go`) как структура:

```go
type slice struct {
    array unsafe.Pointer // Указатель на первый элемент нижележащего массива (underlying array)
    len   int            // Длина: количество инициализированных элементов в слайсе
    cap   int            // Емкость: максимальное количество элементов до аллокации новой памяти
}
```

В Go **всё передается по значению**. При передаче слайса в функцию копируется сам заголовок `slice`, но указатель внутри него продолжает ссылаться на тот же нижележащий массив данных.

---

### Механика работы append и рост емкости

Функция `append` добавляет элементы в слайс, опираясь на значения `len` и `cap`:

1. **`len < cap`:** Элемент записывается в текущий массив. `len` увеличивается. Изменения видны во всех слайсах, разделяющих этот массив.
2. **`len == cap`:** Происходит аллокация нового массива. Старые данные копируются через встроенную функцию `copy`, добавляется новый элемент, а `append` возвращает новый заголовок слайса с измененным указателем. Старый слайс остается привязан к старому массиву.

#### Алгоритм роста емкости (Go 1.23+)

- Если запрашиваемая емкость больше удвоенной старой, выделяется запрашиваемая емкость.
- Если старая емкость `< 256`, она **удваивается** (`newcap = oldcap * 2`).
- Если старая емкость `>= 256`, рост сглаживается по формуле: `newcap = oldcap + (oldcap + 3*256) / 4`. Это обеспечивает плавный переход коэффициента роста от `2.0` к `1.25` по мере увеличения объема данных.

---

### Критические особенности и подводные камни

- **Эффект бокового изменения:** Слайсы, полученные путем среза (`s2 := s1[1:3]`), делят один нижележащий массив. Модификация элементов в `s2` напрямую меняет данные в `s1`. Влияние прекращается только после того, как `append` внутри `s2` спровоцирует выделение нового массива.
- **Append в функциях:** Вызов `append` внутри функции изменяет только локальную копию заголовка слайса (`len` и `cap`). Вызывающий код снаружи функции не увидит добавленных элементов, даже если они записались в тот же массив, так как его собственный заголовок сохранил старое значение `len`. Для изменения слайса в функции необходимо передавать указатель `*[]T` либо возвращать обновленный слайс.
- **Утечки памяти (Memory Leaks):** Короткий срез от огромного слайса (`small := huge[0:2]`) блокирует сборщик мусора (`GC`), так как указатель в `small` удерживает в памяти _весь_ массив `huge`. Для предотвращения утечки необходимо явно копировать данные в новый слайс через `copy()`.

---

### Примеры кода

#### 1. Изоляция после реалокации памяти

```go
package main

import "fmt"

func main() {
	s1 := make([]int, 2, 2) // len: 2, cap: 2
	s1[0], s1[1] = 1, 2

	s2 := s1 // Копируем заголовок. Указатель на массив общий
	s2[0] = 99 // Меняет данные и в s1, и в s2

	s2 = append(s2, 3) // s2 превысил cap и переехал на новый массив
	s2[0] = 88         // Теперь s2 изолирован, s1 остался неизменным

	fmt.Println(s1) // Вывод: [99 2]
	fmt.Println(s2) // Вывод: [88 2 3]
}
```

#### 2. Передача в функцию и потеря len

```go
package main

import "fmt"

func addElement(s []int) {
	s[0] = 5 // Изменение применится к общему массиву
	s = append(s, 6) // Изменит локальный len до 3. Новый элемент наружу не попадет
}

func main() {
	slice := []int{1, 2} // len: 2, cap: 2
	addElement(slice)
	fmt.Println(slice) // Вывод: [5 2] (len по-прежнему 2, элемент 6 недоступен)
}
```

#### 3. Предотвращение утечки памяти через copy()

```go
package main

func getHeader() []byte {
	hugeData := make([]byte, 10<<20) // 10 MB данных из сети

	// ПЛОХО: return hugeData[:2] — 10 MB останутся в памяти навсегда

	// ПРАВИЛЬНО:
	result := make([]byte, 2)
	copy(result, hugeData[:2])
	return result // hugeData успешно уничтожится сборщиком мусора
}
```
