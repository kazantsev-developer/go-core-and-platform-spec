## Спецификация паттернов двух указателей и скользящего окна: Оптимизация обхода структур памяти и линейные фреймворки вычислений на Go

В высоконагруженных бэкенд-системах (Highload) обработка последовательностей данных (массивов, слайсов) часто требует поиска подстрок, сумм или подмножеств элементов, удовлетворяющих заданным критериям. Брутфорс-подходы на базе вложенных циклов порождают квадратичную временную сложность O(N^2), что приводит к критическому росту утилизации CPU. Паттерны Двух указателей (Two Pointers) и Скользящего окна (Sliding Window) оптимизируют вычисления до строго линейного времени O(N), минимизируя накладные расходы на итерации и сохраняя нулевые аллокации в куче (Zero-Allocation).

---

### 1. Архитектурный паттерн двух указателей (Two Pointers)

Паттерн оперирует двумя индексами, которые перемещаются по слайсу данных независимо друг от друга. Это позволяет исключить избыточные повторные сканирования элементов.

#### Сходящиеся указатели (Two Pointers от краев к центру)

Применяется на строго отсортированных массивах данных. Указатель `left` инициализируется на нулевом индексе, указатель `right` — на конечном (`len(arr) - 1`). В зависимости от математического условия на каждой итерации цикла сдвигается либо левый, либо правый индекс. Обход завершается, когда индексы сравниваются.

Промышленный пример: Поиск пары чисел в отсортированном слайсе, сумма которых строго равна целевому значению (`target`). Обычный вложенный цикл потребовал бы времени O(N^2). Сходящиеся указатели решают задачу за один проход O(N) по времени и O(1) по памяти.

```go
package main

// FindTargetPair реализует сходящиеся указатели за O(N) по времени и O(1) по памяти
func FindTargetPair(arr []int, target int) (int, int, bool) {
	left := 0
	right := len(arr) - 1

	for left < right {
		currentSum := arr[left] + arr[right]

		if currentSum == target {
			return left, right, true
		}

		if currentSum < target {
			left++ // Сумма слишком мала, сдвигаем левую границу вправо для увеличения значения
		} else {
			right-- // Сумма слишком велика, сдвигаем правую границу влево для уменьшения значения
		}
	}

	return -1, -1, false // Целевая пара элементов не обнаружена
}
```

#### Скоростные указатели (Fast and Slow Pointers)

Паттерн, при котором оба указателя стартуют с начала массива, но перемещаются с разной скоростью. Быстрый указатель собирает данные или верифицирует условия, а медленный указатель фиксирует позицию перезаписи или валидного состояния данных.

Промышленный пример: In-place дедупликация отсортированного слайса. Медленный указатель фиксирует индекс последней уникальной записи, а быстрый сканирует элементы вперед. Задача решается строго в рамках физических границ исходного массива без аллокаций новых базовых массивов в куче.

```go
// RemoveDuplicates производит дедупликацию за O(N) по времени и O(1) по памяти
func RemoveDuplicates(arr []int) int {
	if len(arr) == 0 {
		return 0
	}

	slow := 0
	for fast := 1; fast < len(arr); fast++ {
		if arr[fast] != arr[slow] {
			slow++
			arr[slow] = arr[fast] // Модификация базового массива на месте без выделения памяти
		}
	}
	return slow + 1 // Возвращает новую фактическую длину уникальной последовательности
}
```

### 2. Паттерн скользящего окна (Sliding Window)

Паттерн «Скользящее окно» преобразует вложенные итерации в обход массива одним непрерывным непрерываемым потоком. Вместо того чтобы пересчитывать подмассив заново на каждом шаге, алгоритм сдвигает границы виртуального окна, добавляя новый элемент справа и выбрасывая старый элемент слева за константное время O(1).

#### Скользящее окно фиксированного размера

Применяется, когда размер подмассива жестко задан условием задачи (например, поиск максимальной суммы элементов в подмассиве длины K).

Промышленный пример: Расчет скользящего среднего или поиск пиковой нагрузки в буфере за фиксированное окно. Алгоритм сначала вычисляет сумму для первых K элементов, а затем в один проход сдвигает окно, вычитая `arr[i-K]` и прибавляя `arr[i]`.

```go
package main

// MaxSumSubarray реализует фиксированное окно за O(N) по времени и O(1) по памяти
func MaxSumSubarray(arr []int, k int) (int, bool) {
	if len(arr) < k || k <= 0 {
		return 0, false
	}

	currentSum := 0
	for i := 0; i < k; i++ {
		currentSum += arr[i]
	}

	maxSum := currentSum

	// Сдвиг окна по всей длине слайса
	for i := k; i < len(arr); i++ {
		currentSum += arr[i] - arr[i-k] // Константный пересчет O(1)
		if currentSum > maxSum {
			maxSum = currentSum
		}
	}

	return maxSum, true
}
```

#### Скользящее окно динамического размера

Применяется, когда размер окна может расширяться или сужаться в зависимости от выполнения динамических условий (например, поиск кратчайшего подмассива, сумма элементов которого больше или равна `target`).

Правая граница окна (`right`) последовательно расширяется в цикле, увеличивая внутреннее состояние. Как только целевое условие выполняется, запускается вложенный цикл, который начинает сжимать левую границу (`left`), пытаясь минимизировать размер окна до тех пор, пока условие остается валидным. Амортизированная временная сложность составляет O(N), так как каждый индекс посещается указателями `left` и `right` максимум дважды за все время работы алгоритма.

```go
// MinSubarrayLen реализует динамическое окно за O(N) по времени и O(1) по памяти
func MinSubarrayLen(target int, arr []int) int {
	minLen := len(arr) + 1
	currentSum := 0
	left := 0

	for right := 0; right < len(arr); right++ {
		currentSum += arr[right]

		// Сужение окна при достижении целевого условия
		for currentSum >= target {
			currentWindowLen := right - left + 1
			if currentWindowLen < minLen {
				minLen = currentWindowLen
			}
			currentSum -= arr[left]
			left++ // Сдвиг левой границы окна
		}
	}

	if minLen > len(arr) {
		return 0
	}
	return minLen
}
```

### 3. Оптимизация динамических окон через битовые маски и хэш-таблицы

При поиске уникальных подстрок (например, самая длинная подстрока без повторяющихся символов) состояние окна необходимо трекать в памяти. Использование структуры `map[byte]int` порождает накладные расходы на выделение памяти в куче (Escape to Heap).

Для ASCII-символов (строки, байты) в высоконагруженных Go-сервисах вместо `map` применяется плоский фиксированный массив-счетчик на стеке:

```go
var pos [256]int // Массив на 256 элементов локализован в стеке горутины с нулевыми аллокациями
```

Это позволяет полностью исключить обращения к системному распределителю памяти и снизить время проверки уникальности символа до долей наносекунды.

### 4. Методология бенчмаркинга паттернов обхода памяти

Верификация эффективности алгоритмов скользящего окна на предмет отсутствия скрытых аллокаций контекста циклов инициализируется через микробенчмарки:

```go
package main

import "testing"

func BenchmarkMaxSumSubarray(b *testing.B) {
	size := 50000
	data := make([]int, size)
	for i := 0; i < size; i++ {
		data[i] = i % 100
	}
	k := 500

	b.ResetTimer()
	for i := 0; i < b.N; i++ {
		_, _ = MaxSumSubarray(data, k)
	}
}
```

Команда запуска бенчмарка в CLI:

```bash
go test -bench=BenchmarkMaxSumSubarray -benchmem
```

Критерий прохождения контроля для enterprise-систем: показатели `B/op` и `allocs/op` должны быть строго равны `0`. Любое значение выше нуля означает дефект escape-анализа и неявное выделение ресурсов в куче хоста, что недопустимо в критических путях выполнения бизнес-логики.
